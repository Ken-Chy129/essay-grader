# 语境术语表 (CONTEXT.md)

> 本文件是**术语表**,只记录领域语言的精确定义,不记录实现细节、方案或待办。
> 当某个词被精确敲定时就地更新这里;有歧义时优先在此澄清。

## 产品一句话

一个**给高中语文老师用**的作文批改辅助工具:老师上传学生手写作文照片,系统帮助
转写、生成评语与评分建议、并沉淀班级层面的数据洞察。学生不直接使用本系统。

## 术语

### 老师 (Teacher)
本系统**唯一的直接用户**。所有界面、交互、产出都面向老师。学生不登录、不访问。

### 作文 (Essay)
一名学生的**一篇**作文。进入系统时 = **1 张或多张有序页面照片**(800 字常跨页,故不强求一图)。
- 多张照片**按顺序拼接**成一篇连续转写稿。
- 每篇作文的批改结果与其**来源图片组**关联;本期**不绑定学生身份**(不维护班级名单/学生档案)。
> 📌 实测修正:原"一张照片=一篇"在真实数据上不成立(测试作文本身就 2 页),改为"一篇=有序页面照片组"。

### 方向校正 (Orientation)
手机照片常旋转躺倒;**方向错会导致模型臆造整篇内容(对批改致命)**,故由**工程 + 人**把关,
**LLM 不参与方向判断**:
- 工程:上传时读 **EXIF** 自动转正(纯 Go);
- 人:每张缩略图可**手动 ↻ 旋转**校正(纯图像)。
> 实测教训:让 LLM 判方向会在判错时编造假作文;改为 EXIF+人工后,LLM 只看正图,转写忠实。详见 ADR-0002。

### 上传/评分解耦
- **上传**:只存图 + EXIF 转正,状态 `uploaded`,**不批改**(每张照片 = 一篇)。
- **确认方向**:老师扫缩略图,歪的点 ↻ 旋转(旋转会重置该篇为待评分)。
- **开始评分**:老师确认后点一次,才统一进入转写+评分。

### 作业 (Assignment)
**一次批改的单元**。对应老师布置的一次作文任务,包含:
- 一个**题目**(见下);
- **一批**学生作文(整班一起交、逐篇批改)。
所有 B(评分一致性)与 D(数据洞察)都以"一次作业"为统计口径。

### 题目 (Prompt)
一次作业的写作要求(含材料作文的材料、写作要求)。是评判**立意/审题**的基准,不可省。
- 支持从照片中**识别**,也支持老师**手动填写/修改**。

### 转写稿 (Transcript)
系统对一篇作文照片识别出的**文字版本**,追求**通顺好读**(因为"字迹难辨"是老师痛点)。
- 不要求逐字保留学生的错别字/漏标点(那些维度本期不评)。
- 老师**可以编辑**转写稿,但这是**可选能力**,不是生成评语前的强制关卡。

### 批改 (Grading)
指**老师在 AI 产出基础上的审核与定稿**,而非 AI 自主判分。
- 流程:照片 → **一次性**生成 [转写稿 + 评语 + 评分建议]→ 老师审核/编辑/采纳。
- AI 只负责**高层评价维度**:立意、结构、语言、内容。
- **本期不评**:错别字、标点、卷面、字数等"低层"维度(AI 不可靠,留给老师看原图)。
- 老师拥有最终决定权。
- 老师编辑转写稿后,评语/评分**不自动更新**;需老师手动触发"重新批改"。
- **异步批处理**:老师整批上传后后台逐篇跑;**批好一篇即可查看一篇**,无需等全批完成。

### 评分参考 (Grading Guide)
老师在建作业时**选填**的一段权威指引:材料解读 / 立意指导 / 思辨要求等。
- 会注入**评分**与**数据洞察(共性问题/选篇)**的 prompt,让 AI 按老师的立意尺度打分,而非泛泛而评。
- 是 B(评分一致性)的关键锚点:把"和谁一致"具体化为"和老师这次的解读一致"。

### 评分 (Score)
- 默认**满分 60**(建作业时可改满分值)。
- AI 产出 = **建议总分 + 分维度评价**,显著标注"AI 建议,老师可改"。
- 基准 = **通用高考作文评分框架(基准甲)**;本期**不做**按某位老师打分习惯的个性化校准(基准乙)。

### 评分维度 (Rubric Dimensions)
评分所依据的维度必须**对老师明示**,且**老师可修改调整**(增/删/改名/调整)。
- 默认内置维度:**立意、结构、语言、内容**(默认各 15 分,合计 60)。
- 每个维度**各带分值配额**,AI 给每维度打分,**总分 = 各维度分数之和**(可解释、可追溯;老师改某维度分,总分联动)。
- 调整粒度:**全局一个"默认维度模板"**,每次作业可**临时覆盖**。

### 评语 (Comment)
单篇作文的文字评价,由 AI 生成初稿、老师可改。形态:
- **总评** 2~4 句;
- **每个维度一句具体点评**(点出问题 + 一句改进方向),与该维度的分数对应。
- **不做**面向学生的逐句精批(那是已排除的 C)。

### 数据洞察 (Insights)
以**一次作业**为口径,在**整批批改完成后**统一产出(与"逐篇可见"是不同阶段)。本期产出:
1. **分数分布**:平均/最高/最低/分布。
2. **维度短板**:哪个维度全班普遍最弱。
3. **共性问题**:AI 纵览全班评语后二次归纳出的反复出现的问题(D 的核心价值)。
4. **讲评选篇**:自动挑出"可作范文 / 典型问题"的篇目,供作文讲评课使用。
> 价值优先级:**A(省时间) > B(评分) > D(洞察)**;D 依赖前两步质量,落地排在 A、B 跑通之后。

### 定位与边界(本期)
- 形态:**单个老师的私人工具**。本期**不做**账号体系、多租户、未成年人数据合规。
- 优先级:先验证"AI 批改老师认不认",而非规模化。数据模型可为将来留余地,但不提前搭多租户。

### 内核范围
- **A 省时间**:减少老师人肉批改的时间成本。(主)
- **B 评分一致性**:对齐高考/教学评分标准,减少打分漂移。
- **D 数据洞察**:**本期轻量化**——只做"一次作业里这批作文的共性问题",**不细到每个学生**的成长曲线(后续迭代)。
- **(排除) C 面向学生的逐句精批**:不在范围内。

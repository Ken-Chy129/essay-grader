# 0001 — 批改采用「转写 / 评判」两阶段管线,而非多模态一把梭

- 状态:已接受 (Accepted)
- 日期:2026-06-30

## 背景 (Context)

输入是高中生**手写作文照片**(本期约定一图一篇)。已实测:Claude Opus 4.8 直接
"看图 → 出评语/评分"(一把梭)效果可用。因此一个自然的诱惑是:让多模态模型一步到位。

但产品已确定的几个需求,对管线形态提出了额外约束:
- 路线 1:**转写稿是一个对老师可见、可编辑的产物**(为解决"字迹难辨"痛点)。
- 老师**编辑转写稿后可手动"重新批改"**。
- D 数据洞察需要**纵览全班评语做二次归纳**。

## 决策 (Decision)

批改拆成**两个阶段**:

1. **转写阶段**:多模态模型(Claude Opus 4.8)将**图片 → 转写稿**(通顺好读,不强求保留错别字)。
2. **评判阶段**:文本模型基于 **(转写稿 + 题目 + 评分维度) → 评语 + 分维度评分**。

第一顺位模型统一用 **Claude Opus 4.8**(经 llm-proxy 网关调用),后续可按成本/效果替换第①步
(如换更便宜的中文视觉模型)而不影响第②步。

## 后果 (Consequences)

好处:
- 转写稿成为**显式产物**,天然满足"可查看/可编辑"。
- **"重新批改"只需重跑第②步**(不必重新识图)→ 更快、更省、对编辑后的文本忠实。
- D 的二次归纳是**纯文本**操作,与逐篇批改解耦。
- 两步可**分别替换模型/调 prompt**,演进成本低。

代价:
- 比一把梭多一次模型调用与一层中间状态。
- 第②步的质量受第①步转写质量影响(转写错 → 评判基于错文本);靠"老师可编辑转写稿 + 重新批改"兜底。

## 被否决的备选 (Alternatives)

- **多模态一把梭(图 → 直接评语/评分)**:实测可用且调用更少,但无法天然产出可编辑的转写稿,
  "重新批改"必须重新识图,且 D 仍需另起一步——综合下来反而更绕。故否决。
